# Design System / Taste Gate

Taste Skill may be used as a frontend design reference, especially for:
- premium layout,
- hierarchy,
- typography,
- spacing,
- motion,
- avoiding generic AI UI,
- redesign audits,
- complete output.

Repo truth still wins:

1. `design.md` defines design doctrine.
2. `tokens.css` defines design tokens.
3. `design/taste_profile.yaml` defines taste mode and intensity.
4. `scripts/design-system-lint.mjs` blocks raw design values.
5. UI claims require screenshot or video evidence.

Do not import animation or UI dependencies without verifying they are installed
and approved for the package.

Do not leave placeholder comments, skipped blocks, or unfinished UI.
