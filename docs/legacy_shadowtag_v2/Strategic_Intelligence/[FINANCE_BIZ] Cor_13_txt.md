2. Technical Security Controls

Protect the actual artifacts: source code, models, datasets, and NVIDIA-optimized build.

a) Source Code (GitHub / Claude)
• Private repos only → no public forks.
• Enable branch protection + signed commits.
• Require 2FA + hardware keys (YubiKeys) for all devs.
• Use GitHub Advanced Security to scan for secrets and license violations.
• Create a separate repo for deployment YAMLs to isolate infra knowledge.

b) Model Artifacts
• Store trained weights only in encrypted S3 buckets or a private registry.
• Use KMS keys with strict IAM roles (least privilege).
• Hash each TensorRT/ONNX export and store checksums in GitHub.
• Enable model signing before deploying to Triton.

⸻

c) Datasets
• PII always anonymized + encrypted at rest and in transit.
• Adopt GPUDirect Storage with signed binary drivers from NVIDIA only.
• Separate “raw ingest” zone vs “training-ready zone” with access control policies.
