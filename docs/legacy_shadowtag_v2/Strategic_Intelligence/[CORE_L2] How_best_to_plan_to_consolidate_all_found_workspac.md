# Antigravity Operations Guide 🚀

## Quick Start Commands

The **Antigravity Mission Launcher** (`python3 pnkln_mission_start.py`) provides an interactive menu to run these commands.

| Command                                    | Description            | What it does                                                                                                                                                                                         |
| :----------------------------------------- | :--------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`npm run dev`**                          | **Multi-Model Router** | Starts the central API Gateway. This routes requests to different AI models (Gemini, Claude, Grok, etc.) based on cost, speed, and capability.                                                       |
| **`npm run triple:pass`**                  | **ACE Workflow**       | Launches the "Agentic Context Engineering" workflow. <br>1. **Generates** code. <br>2. **Explains** the code (reverse engineering). <br>3. **Critiques/Fixes** the code via an opposing model chain. |
| **`python -m ingestion.moondream_ingest`** | **Vision Ingestion**   | Uses the Moondream VLM (Vision Language Model) to analyze images/video, extract metadata, and feed the intelligence pipeline.                                                                        |
| **`python -m computer_use.agent`**         | **Computer-Use Agent** | Starts an autonomous agent capable of interacting with the OS (GUI/Terminal) to perform complex tasks.                                                                                               |

## 🔐 Safety Framework

Located in `safety/`, this framework ensures all AI operations are governed, risk-assessed, and monitored.

### 1. Item Definition (`safety/item_definition.md`)

Defines **what** is being built and governed.

- **Service Name**: Identity of the component.
- **Criticality**: Impact level (Low, Medium, High, Critical).
- **Owner**: Responsible team/person.

### 2. Risk Register (`safety/risk_register.yaml`)

Tracks potential failure modes and mitigations.

- **Risk**: Description of the bad outcome (e.g., "Agent deletes production database").
- **Likelihood/Severity**: Risk scoring.
- **Mitigation**: Controls in place (e.g., "Read-only DB credentials").

### 3. Service Level Objectives (`safety/monitoring/slos.json`)

Defines the reliability and safety targets.

- **Availability**: Uptime target (e.g., 99.9%).
- **Latency**: Max acceptable delay.
- **Safety Checks**: % of operations required to pass safety gates (Target: 100%).
