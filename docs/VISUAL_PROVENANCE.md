# TACSOP 7 — Visual Provenance & Native Image Generation Ban

> **Status**: ACTIVE (2026-04-25)
> **Enforcement**: Behavioral invariant + Progressive Disclosure skill
> **Skill**: `.agents/skills/ban-native-image-gen/SKILL.md`
> **Risk**: #94 in `RISK_REGISTER.md`

---

## 1. Problem Statement

AI agents (Antigravity, Claude, Cursor, Cline) have access to a native `generate_image`
tool that produces images with no watermark, no metadata, and no provenance chain.
Images produced by this tool:

- Cannot be traced to their generation model or prompt
- Cannot be verified as AI-generated by downstream consumers
- Violate EU AI Act Article 52 (AI-generated content disclosure requirements)
- Cannot be audited in our Firestore-based media asset pipeline
- Risk brand confusion when mixed with human-created brand assets

## 2. Ban Scope

### Absolute Prohibitions

| Action | Tool | Status |
|--------|------|--------|
| Native image generation in any agent | `generate_image` | **BANNED** |
| Saving generated images directly to project filesystem | `generate_image` → `write_to_file` | **BANNED** |
| Using generated images in HTML/CSS `src` or `url()` | All agents | **BANNED** |
| Claiming generated images are "stock" or "placeholder" | All agents | **BANNED** |

### Authorized Alternatives

| Purpose | Authorized Tool | Provenance |
|---------|----------------|------------|
| Static UI images | Stitch MCP `generate_screen_from_text` | Stitch project metadata |
| Video backgrounds | Veo 3.1 via Gemini API | Gemini API audit log |
| Product photos | Nano Banana 2 via Gemini API | Gemini API audit log |
| Design exploration | Stitch MCP `generate_variants` | Stitch project metadata |
| Placeholder UI | CSS gradient/SVG patterns | Deterministic (no AI) |

## 4. DESIGN.md Integration

When using Stitch MCP or the DESIGN.md specification for visual assets:

```yaml
# In .stitch/DESIGN.md or project DESIGN.md
visuals:
  hero_background:
    type: video_loop
    generator: veo-3.1
    prompt: "Abstract technological grid, deep navy and gold"
    provenance: gemini-api-audit-log
  
  product_screenshot:
    type: static_image
    generator: stitch-mcp
    project_id: "12345678"
    screen_id: "abcdef"
    provenance: stitch-project-metadata
  
  icon_set:
    type: svg
    generator: manual
    source: "Material Design Icons"
    provenance: open-source-license
```

## 5. Media Asset Pipeline

All visual assets in production MUST flow through:

```
Authorized Generator (Stitch/Veo/Nano Banana)
  → Firestore `media_assets` collection
  → Fields: { generator, prompt_hash, created_at, provenance_type, audit_url }
  → Firebase Storage (signed URL)
  → CDN delivery with `X-Provenance` header
```

## 7. Audit & Compliance

### EU AI Act Article 52 Compliance
- All AI-generated visual content MUST be labeled as such
- Metadata MUST identify the generating model
- Provenance chain MUST be auditable by request

### Internal Audit Trail
- Every visual asset has a Firestore document in `media_assets`
- Monthly audit: scan all `<img>`, `background-image`, and `<video>` in deployed sites
- Cross-reference against `media_assets` collection for orphaned assets
