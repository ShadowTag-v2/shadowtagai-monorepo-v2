## 1. Firebase App Check — reCAPTCHA Enterprise Setup

**Status:** Code wired (`src/lib/firebase.ts`) — needs Console registration.

### Steps:
1. Go to [Firebase Console → App Check](https://console.firebase.google.com/project/shadowtag-omega-v4/appcheck)
2. Click **Register** next to the HeadFade web app (`1:767252945109:web:f05bd5fa9c87a7dfcb2a5c`)
3. Select **reCAPTCHA Enterprise** as the provider
4. Copy the generated **site key**
5. Set `NEXT_PUBLIC_RECAPTCHA_ENTERPRISE_SITE_KEY=<site_key>` in your environment
6. Rebuild and redeploy
7. Return to App Check and **enable enforcement** for:
   - ✅ Cloud Firestore
   - ✅ Cloud Storage (if used)
   - ✅ Authentication

> **Warning:** Don't enable enforcement until the new build with the real site key is deployed, or all existing clients will be blocked.

## 2. Apple OAuth Provider — Firebase Auth

**Status:** Google OAuth is live. Apple OAuth needs Apple Developer account.

### Steps:
1. Go to [Apple Developer → Certificates, Identifiers & Profiles](https://developer.apple.com/account/resources)
2. Register a new **Service ID**:
   - Identifier: `com.headfade.signin`
   - Enable "Sign In with Apple"
   - Configure domains: `shadowtag-omega-v4.firebaseapp.com` and `headfade.web.app`
   - Return URL: `https://shadowtag-omega-v4.firebaseapp.com/__/auth/handler`
3. Generate a **Key** for Sign In with Apple:
   - Download the `.p8` key file
   - Note the Key ID
4. Go to [Firebase Console → Authentication → Sign-in method](https://console.firebase.google.com/project/shadowtag-omega-v4/authentication/providers)
5. Enable **Apple** provider:
   - Service ID: `com.headfade.signin`
   - Apple Team ID: (from Apple Developer account)
   - Key ID: (from step 3)
   - Private Key: (paste contents of `.p8` file)
6. Upload the private key to GCP Secret Manager:
   ```bash
   gcloud secrets create apple-signin-private-key \
     --data-file=AuthKey_XXXXXXXXXX.p8 \
     --project=shadowtag-omega-v4
   ```

## 3. HSTS Preload Submission

**Status:** HSTS header added to `firebase.json` with `preload` directive.

### Steps:
1. Verify the header is deployed:
   ```bash
   curl -sI https://headfade.web.app | grep -i strict
   # Expected: Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
   ```
2. Go to [hstspreload.org](https://hstspreload.org)
3. Enter `headfade.web.app` (and/or `headfade.com` when custom domain is active)
4. Submit for inclusion in browser HSTS preload lists

> **Note:** HSTS preload is **permanent** — removal takes months. Only submit after confirming HTTPS works perfectly on all subdomains.

## 4. Custom Domain DNS — headfade.com

**Status:** Firebase Hosting custom domain needs DNS TXT verification.

### Steps:
1. Go to [Firebase Console → Hosting](https://console.firebase.google.com/project/shadowtag-omega-v4/hosting)
2. Click **Add custom domain** → enter `headfade.com`
3. Firebase will provide TXT verification records
4. In your DNS provider (Namecheap/Cloudflare/etc):
   - Add the TXT record for domain verification
   - After verification, add the A records Firebase provides:
     ```
     A    @    151.101.1.195
     A    @    151.101.65.195
     ```
   - Add CNAME for www:
     ```
     CNAME    www    headfade.web.app.
     ```
5. Wait for SSL provisioning (can take 1-24 hours)
6. **Important:** If using Cloudflare, set DNS to **DNS Only** (gray cloud), not **Proxied** (orange cloud)

### Optional: Email forwarding
If you want `contact@headfade.com`:
- Add MX records pointing to your email provider
- Or use Cloudflare Email Routing (free)
