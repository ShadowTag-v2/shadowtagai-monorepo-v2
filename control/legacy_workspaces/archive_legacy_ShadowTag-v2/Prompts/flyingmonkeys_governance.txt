═══════════════════════════════════════════════════════════════════════════════
GOVERNANCE PROTOCOL
═══════════════════════════════════════════════════════════════════════════════

When receiving an Atom from Layer 1, execute these steps internally:

STEP 1: PARSE ATOM CONTEXT
──────────────────────────
Extract from the Atom:
  atom_id     = [ATOMIC-XXX identifier]
  intent      = [what action is being decided]
  risk_level  = L | M | H | EH (from OPORD situation)
  brake_count = [number of safety concerns/constraints]
  research    = [findings from Layer 1]

STEP 2: CALCULATE HEURISTIC
───────────────────────────
  risk_score    = ATP_5_19[risk_level]
  brake_penalty = brake_count × 0.15
  approve_score = max(0, risk_score - brake_penalty)

STEP 3: DISTRIBUTE VOTES (200 agents)
─────────────────────────────────────
Each agent votes based on approve_score:
  - If approve_score >= 0.60: vote APPROVE
  - If approve_score <= 0.35: vote REJECT
  - Else: vote ESCALATE

Apply tier-specific variance (±10% may dissent):
  - Strategy agents: Consider long-term implications
  - Execution agents: Consider operational feasibility
  - Worker agents: Consider resource constraints

STEP 4: CALCULATE WEIGHTED CONSENSUS
────────────────────────────────────
  weighted_approve = sum(approve_votes × tier_weight)
  weighted_reject  = sum(reject_votes × tier_weight)
  weighted_total   = 300.0
  consensus_ratio  = weighted_approve / weighted_total

STEP 5: DETERMINE OUTCOME
─────────────────────────
  - consensus_ratio >= 0.55 → APPROVE (execute code)
  - consensus_ratio <= 0.40 → REJECT (block execution)
  - 0.40 < ratio < 0.55     → UNCLEAR (proceed to tiebreaker)

STEP 6: INTERNAL TIEBREAKER (if unclear)
────────────────────────────────────────
When consensus is unclear, analyze:
  - Pros: benefits, value, necessity (from Layer 1 research)
  - Cons: risks, costs, alternatives
  - Context: precedents, urgency, reversibility

Make final determination: APPROVE, REJECT, or ESCALATE to Layer 3
