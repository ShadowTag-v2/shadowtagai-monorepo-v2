### Description of the bug

I am running VS Code Insiders.

On the README of this repository It is explained to run the Chrome MCP server installation using `code --add-mcp '{"name":"chrome-devtools","command":"npx","args":["chrome-devtools-mcp@latest"]}'` which is straightforward and produces a simple 4 line MCP server configuration in mcp.json:

```
    "chrome-devtools": {
      "command": "npx",
      "args": ["chrome-devtools-mcp@latest"],
      "type": "stdio"
    }
```

After doing so and having tested out the MCP server which seemingly works okay (with unrelated bugs) I saw that Chrome DevTools MCP is also available to install through the VS Code extensions marketplace. I would have expected it to be the same installation so that I would not have the option to install it again.

I then went on to install the marketplace extension and that produced MANY more lines in the mcp.json file:

```
"chromedevtools/chrome-devtools-mcp": {
      "type": "stdio",
      "command": "npx",
      "args": [
        "chrome-devtools-mcp@latest",
        "--browserUrl",
        "${input:browser_url}",
        "--headless",
        "${input:headless}",
        "--isolated",
        "${input:isolated}",
        "--channel",
        "${input:chrome_channel}"
      ],
      "gallery": "https://api.mcp.github.com/2025-09-15",
      "version": "0.0.1-seed"
  },
  "inputs": [
    {
      "id": "browser_url",
      "type": "promptString",
      "description": "Optional: connect to an already-running Chrome (remote debugging / port-forward). Example: http://127.0.0.1:9222",
      "password": false
    },
    {
      "id": "headless",
      "type": "promptString",
      "description": "Run Chrome headless (true/false). Default: false",
      "password": false,
      "default": "false"
    },
    {
      "id": "isolated",
      "type": "promptString",
      "description": "Use a temporary user-data-dir (true/false). Default: false",
      "password": false,
      "default": "false"
    },
    {
      "id": "chrome_channel",
      "type": "promptString",
      "description": "Chrome channel: stable | canary | beta | dev (default: stable).",
      "password": false,
      "default": "stable"
    }
  ]
```

Now to my questions. Which is the correct/preferred way to install this MCP server? What will give me the best feature set and the most recent updates as new versions of the MCP server is released?

In the simple one I am struggling to see which version is actually installed as it does not even show the version on server start (maybe it should?).

### Reproduction

Install manually and through the marketplace.

### Expectation

Less confusion on which is the correct package/server to install, and that differences of features, if any, are explained in those instructions.

I think it's as easy to install through the marketplace as to copy/paste in the one-liner command line execution, so it does not really matter if it's one of the other way, but it's confusing to have both options and that they produce different output.
