# Monorepo Best Practices Rules

## Directory Structure Enforcement

- Application code MUST reside in `apps/<app_name>/`
- Shared libraries MUST reside in `libs/<lib_name>/`
- Infrastructure code MUST reside in `infra/`
- Developer tools MUST reside in `tools/`
- Build scripts MUST reside in `scripts/`
- Third-party vendored code MUST reside in `third_party/`

## Build System

- Every directory with source code MUST have a `BUILD.bazel` file
- Dependencies between apps MUST go through `libs/` — no direct app-to-app imports
- All external dependencies MUST be declared in `MODULE.bazel`
- Python dependencies MUST be locked via `requirements.txt` (generated by Bazel)

## Security

- No secrets in source code — use GCP Secret Manager
- No `.env` files — use `scripts/load_mcp_secrets.sh`
- All API endpoints MUST validate authentication
- CORS policies MUST be explicit (no `*` in production)
- CSP headers MUST be set for all hosting targets

## Code Quality

- Python: `ruff check --fix` + `ruff format` after every edit
- TypeScript/JavaScript: `biome check --fix` after every edit
- Go: `gofmt` + `golangci-lint`
- Rust: `cargo fmt` + `cargo clippy`
- No wildcard imports
- No unused imports (enforced by linters)

## Git Workflow

- All commits use conventional commit format
- PRs require at least one approval
- CI must pass before merge
- Force-pushes to `main` are forbidden
- Branch naming: `feat/`, `fix/`, `chore/`, `docs/`
