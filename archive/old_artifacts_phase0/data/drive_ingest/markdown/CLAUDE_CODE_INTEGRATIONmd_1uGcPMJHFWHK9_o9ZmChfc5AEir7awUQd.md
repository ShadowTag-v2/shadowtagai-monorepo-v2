# Claude Code Memory Integration

**Persistent memory for Claude Code sessions using consensus archive patterns**

## Overview

This integration extracts patterns and learnings from your multi-LLM consensus queries and makes them available to Claude Code sessions via a memory file.

**Benefits:**
- ✅ Claude Code remembers your preferences across sessions
- ✅ Auto-learns from your consensus queries
- ✅ Syncs across devices via GitHub
- ✅ Zero manual maintenance

---

## Architecture

```
Consensus Archive (~/.consensus_archive.db)
    ↓ (pattern extraction via Gemini Flash)
~/.claude-code/memory.md
    ↓ (auto-loaded on Claude Code startup)
Claude Code Session
    ↓ (commit to git)
GitHub Repository
    ↓ (sync across devices)
All Your Machines
```

**Flow:**
1. Run consensus queries → saved to archive
2. Extract patterns with `python claude_code_memory.py sync`
3. Memory written to `~/.claude-code/memory.md`
4. Claude Code auto-loads memory on startup
5. Commit to GitHub for cross-device sync

## Pattern Extraction Details

### With Gemini Flash (Recommended)

When `GOOGLE_API_KEY` is set:

1. **Reads recent transcripts** from consensus archive
2. **Analyzes with Gemini 2.0 Flash**:
   - Identifies coding patterns
   - Extracts preferences
   - Summarizes domain knowledge
   - Detects recurring themes
3. **Generates concise memory** (max 500 words)
4. **Cost:** ~$0.0001 per sync (negligible)

**Why Gemini Flash?**
- Fast (< 1 second)
- Cheap ($0.075 per 1M tokens)
- Good at pattern recognition
- Matches Ultrathink 40% allocation

### Without Gemini (Fallback)

If no `GOOGLE_API_KEY`:

1. Generates basic summary:
   - Total query count
   - Common tags
   - Recent query titles
2. Less intelligent but still useful
3. Zero cost

**Recommendation:** Use Gemini for best results

## Integration with Claude Code

### Auto-Load Memory on Startup

**Method 1: Startup Hook**

Create `~/.claude-code/hooks/startup.sh`:

```bash
#!/bin/bash
# Auto-load memory into Claude Code session

if [ -f ~/.claude-code/memory.md ]; then
    echo "Loading persistent memory from consensus system..."
    cat ~/.claude-code/memory.md
fi
```

Make executable:
```bash
chmod +x ~/.claude-code/hooks/startup.sh
```

**Method 2: Include in System Prompt**

If using Claude Code's system prompt configuration:

```markdown
You are Claude Code with persistent memory.

At the start of each session, load context from:
~/.claude-code/memory.md

This contains patterns extracted from my recent AI consensus queries.
```

## Troubleshooting

### Memory file not updating

```bash
# Check if archive has transcripts
sqlite3 ~/.consensus_archive.db "SELECT COUNT(*) FROM transcripts;"

# Force sync
python claude_code_memory.py sync 30
```

### Gemini extraction failing

```bash
# Check API key
echo $GOOGLE_API_KEY

# Use fallback mode (no Gemini)
unset GOOGLE_API_KEY
python claude_code_memory.py sync
```

### Claude Code not loading memory

```bash
# Verify file exists
cat ~/.claude-code/memory.md

# Check file permissions
ls -la ~/.claude-code/memory.md

# Re-configure startup hook
cat > ~/.claude-code/startup.md << 'EOF'
$(cat ~/.claude-code/memory.md)
EOF
```
