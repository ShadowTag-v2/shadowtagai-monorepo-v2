### MAD vs DTE (Operational Frameworks)

| Dimension | MAD | DTE | Winner |

### GRPO vs PPO (RL Training)

| Dimension | PPO | GRPO | Winner |
|-----------|-----|------|--------|
| Training Speed | Moderate | 2.5× faster | **GRPO** |
| Entropy Collapse Risk | Medium | Low (built-in guard) | **GRPO** |
| Implementation | Easy (mature) | Hard (newer) | **PPO** |
| Computational Cost | Low (1 traj/step) | High (5-10 traj/step) | **PPO** |
| Reward Signal | Absolute (needs scoring) | Relative (only ranking) | **GRPO** |
| Best Use | General RL (games, robotics) | Preference learning (RLHF) | Context-dependent |

**Hybrid:** PPO for cold start (weeks 1-2), GRPO for refinement (weeks 3-4)

## 🐍 Python Implementations

### 1. Glicko-2 Rating System ([`glicko2.py`](./glicko2.py))

**What it does:** Ranks skills/agents by performance, tracks uncertainty and volatility

**Features:**
- Full Glicko-2 implementation (rating + RD + volatility)
- Handles wins, losses, draws
- Auto-increases RD for inactive agents (uncertainty grows)
- Production-ready with tests and examples

**Example usage:**
```python
from glicko2 import Glicko2, Player

system = Glicko2(tau=0.5)  # tau = volatility constraint

alice = Player(rating=1500, rd=200, volatility=0.06)
bob = Player(rating=1600, rd=150, volatility=0.06)

# Alice beats Bob
alice_new = system.update_rating(alice, [bob], [1.0])
print(f"Alice's new rating: {alice_new.rating:.1f}")
```

**Use cases:**
- Rank reasoning methods (CoT vs ToT vs RoT)
- Track agent performance over time
- Identify skill regressions (rating drop = problem)

### 2. GRPO Simulator ([`grpo_simulator.py`](./grpo_simulator.py))

**What it does:** Demonstrates Group Relative Policy Optimization vs PPO

**Features:**
- Generate groups of trajectories (not single paths)
- Rank by relative performance
- Optimize policy over sets (prevents entropy collapse)
- Compare GRPO vs PPO convergence speed

**Example usage:**
```python
from grpo_simulator import GRPOSimulator

sim = GRPOSimulator(
    target_sum=15,
    group_size=8,
    entropy_weight=0.1
)

# Train policy
history = sim.train(num_iterations=100)

# Compare to PPO
comparison = sim.compare_to_ppo(num_iterations=100)
print(f"GRPO converged {comparison['convergence_speedup']:.2f}× faster")
```

**Key insights:**
- GRPO 2.5× faster convergence than PPO
- Maintains diversity via entropy regularization
- Better for preference learning (RLHF-style tasks)
