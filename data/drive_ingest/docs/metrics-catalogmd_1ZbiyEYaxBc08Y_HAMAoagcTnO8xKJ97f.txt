# Metrics Catalog: Standard KPIs by Component Type

**Purpose:** Standardized metrics for SHADOWTAGAI Core Stack™ components
**Usage:** Select metrics based on component type, set targets, track progress

---

## Component Type Matrix

| Type | Primary Goal | Key Metrics | Cost Model |
|------|--------------|-------------|------------|
| **Collection** | Gather data efficiently | Volume, diversity, quality, runtime | $/item |
| **Validation** | Enforce policies accurately | Latency, accuracy, throughput | $/operation |
| **Transformation** | Process data reliably | Speed, completeness, schema compliance | $/record |
| **Delivery** | Present data effectively | Response time, availability, satisfaction | $/user |

---

## Collection/Ingestion Components

**Use for:** Web scrapers, API aggregators, data ingestion pipelines

### Volume Metrics

| Metric | Definition | Target (Example) | Measurement |
|--------|------------|------------------|-------------|
| **Daily Items Ingested** | Count of records collected per day | ≥5,000 items/day | Counter in logs |
| **Source Diversity** | Number of distinct data sources | ≥10 sources | Unique source count |
| **Coverage Rate** | % of target sources successfully scraped | ≥95% | (Successful sources / Total sources) × 100 |

### Quality Metrics

| Metric | Definition | Target (Example) | Measurement |
|--------|------------|------------------|-------------|
| **Relevance Score** | How well items match use case | ≥8/10 | ML model or manual rating |
| **Completeness** | % of required fields populated | ≥95% | (Populated fields / Total fields) × 100 |
| **Timeliness** | Lag between publish time and ingestion | ≤6 hours | Timestamp diff |
| **Duplicate Rate** | % of items that are duplicates | ≤5% | Deduplication hit rate |

### Efficiency Metrics

| Metric | Definition | Target (Example) | Measurement |
|--------|------------|------------------|-------------|
| **Runtime** | Time to complete ingestion job | ≤45 min/night | Job duration |
| **Cost per Item** | Total cost / items ingested | ≤$0.02/item | Monthly cost / monthly items |
| **Error Rate** | % of sources that fail | ≤10% | (Failed sources / Total sources) × 100 |

### Tiering Metrics (if applicable)

| Tier | Definition | Target Distribution | Value |
|------|------------|---------------------|-------|
| **Tier 1** | High-value, exclusive sources | 20% of items | Premium content |
| **Tier 2** | Medium-value, mainstream sources | 50% of items | Standard content |
| **Tier 3** | Low-value, commodity sources | 30% of items | Filler content |

### Ethical Compliance Metrics

| Metric | Definition | Target | Measurement |
|--------|------------|--------|-------------|
| **Robots.txt Compliance** | % of requests respecting robots.txt | 100% | Compliance checker |
| **Rate Limit Adherence** | Requests per second vs. limit | ≤1 req/2 sec | Request timing logs |
| **Opt-Out Requests** | Sites requesting exclusion | <5/month | Support ticket count |

## Validation/Enforcement Components

**Use for:** Policy enforcers, compliance validators, request filters

### Performance Metrics

| Metric | Definition | Target (Example) | Measurement |
|--------|------------|------------------|-------------|
| **Latency (p50)** | Median response time | ≤30ms | Percentile from logs |
| **Latency (p95)** | 95th percentile response time | ≤60ms | Percentile from logs |
| **Latency (p99)** | 99th percentile response time | ≤90ms | Percentile from logs |
| **Throughput** | Requests per second | ≥500 rps | Request counter |

### Accuracy Metrics

| Metric | Definition | Target (Example) | Measurement |
|--------|------------|------------------|-------------|
| **Precision** | TP / (TP + FP) | ≥98% | True Positives / (TP + False Positives) |
| **Recall** | TP / (TP + FN) | ≥98% | True Positives / (TP + False Negatives) |
| **F1 Score** | 2 × (Precision × Recall) / (Precision + Recall) | ≥0.98 | Harmonic mean |
| **Block Rate** | % of violations caught | ≥98% | Blocked / Total violations |
| **False Positive Rate** | FP / (FP + TN) | ≤2% | False alarms / Total negatives |
| **False Negative Rate** | FN / (FN + TP) | ≤2% | Missed violations / Total positives |

### Reliability Metrics

| Metric | Definition | Target (Example) | Measurement |
|--------|------------|------------------|-------------|
| **Uptime** | % of time service is available | ≥99.9% | (Total time - Downtime) / Total time |
| **Error Rate** | % of requests that fail | ≤0.1% | Errors / Total requests |
| **Test Coverage** | % of code covered by tests | ≥98% | Coverage tool (pytest-cov, Jest) |

### Cost Metrics

| Metric | Definition | Target (Example) | Measurement |
|--------|------------|------------------|-------------|
| **Cost per Validation** | API call cost + infra cost per request | ≤$0.0006 | Total cost / Total validations |
| **Monthly Operational Cost** | Total infrastructure + API costs | $5,800/mo (at 10M validations) | Billing dashboard |

## Metric Selection Guide

### Step 1: Identify Component Type

Refer to **Component Type Matrix** above.

### Step 2: Select Core Metrics (Minimum)

For each type, choose:
- **3-5 Performance metrics** (e.g., latency, throughput)
- **2-3 Quality metrics** (e.g., accuracy, completeness)
- **1-2 Cost metrics** (e.g., cost per operation)

### Step 3: Add Optional Metrics

Based on specific needs:
- **Tiering** (if data has value tiers)
- **Ethical Compliance** (for crawlers)
- **User Experience** (for user-facing components)

### Step 4: Set Targets

Use these benchmarks:

| Metric Category | Good | Great | Exceptional |
|-----------------|------|-------|-------------|
| **Latency (p99)** | ≤200ms | ≤100ms | ≤50ms |
| **Throughput** | ≥100 rps | ≥500 rps | ≥1,000 rps |
| **Accuracy (F1)** | ≥0.95 | ≥0.98 | ≥0.99 |
| **Uptime** | ≥99% | ≥99.9% | ≥99.99% |
| **Cost Efficiency** | Linear scaling | Sublinear scaling | Logarithmic scaling |

### Step 5: Implement Tracking

**Tools:**
- **Logs:** Stackdriver (GCP), CloudWatch (AWS)
- **Metrics:** Prometheus, Grafana, Datadog
- **APM:** New Relic, Dynatrace, Elastic APM
- **Custom:** BigQuery for ad-hoc analysis

**Best Practice:** Export metrics to BigQuery for long-term trend analysis.

## Metric Thresholds for AiYouJR Gates

When using component metrics in AiYouJR strategic gates:

### Purpose Gate
- Use **relevance score** (is data aligned with strategic goals?)
- Use **tier distribution** (are we collecting high-value data?)

### Reasons Gate (ROI Calculation)
- Use **cost per operation** (investment calculation)
- Use **volume metrics** (revenue potential)
- Use **efficiency metrics** (operational leverage)

**Example ROI Calculation:**
```
Component: Ingestion Layer
Daily items: 5,000
Cost per item: $0.02
Monthly cost: 5,000 × 30 × $0.02 = $3,000

Revenue potential (AM Briefing):
- Subscribers: 500
- Price: $20/month
- Revenue: $10,000/month

ROI = ($10,000 - $3,000) / $3,000 = 2.33× ✅ (≥3× target needs optimization)
```

### Brakes Gate (Risk Assessment)
- Use **uptime** (reliability check)
- Use **error rate** (quality floor)
- Use **blast radius** from Integration Analysis (impact assessment)

**Example Kill-Switch:**
```
If:
- Uptime <99% for 7 days OR
- Error rate >5% for 3 days OR
- Cost per item >$0.03 (50% over budget)
→ Trigger: Pause ingestion, investigate, rollback if needed
```

## Metric Evolution: Lifecycle Stages

Metrics should evolve as components mature:

### Stage 1: Development (Pre-Prod)
**Focus:** Functional correctness
- Test coverage ≥98%
- Schema compliance 100%
- No hardcoded secrets

### Stage 2: Alpha (Internal Testing)
**Focus:** Performance baseline
- Latency benchmarks established
- Throughput limits identified
- Cost per operation measured

### Stage 3: Beta (Limited Production)
**Focus:** Quality & reliability
- Accuracy metrics (FP/FN rates)
- Uptime ≥99%
- User feedback collected

### Stage 4: Production (General Availability)
**Focus:** Efficiency & scale
- Cost optimization (ROI ≥3×)
- Sublinear scaling achieved
- SLA compliance (99.9% uptime)

### Stage 5: Mature (>6 Months in Prod)
**Focus:** Strategic value
- Revenue contribution tracked
- Competitive advantage measured (e.g., faster than competitors)
- Long-term trends analyzed (seasonality, growth)
