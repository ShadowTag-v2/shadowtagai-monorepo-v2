## Part 2: Cloud Provider Deep Dive

### GCP vs AWS vs Azure: The Real Costs

**Scenario**: Deploying CineVerse streaming infrastructure for 1M concurrent users

#### Google Cloud Platform (GCP)

**Strengths**:
- Best GPU availability (A100, H100)
- Superior network (private fiber backbone)
- Best-in-class Kubernetes (GKE autopilot)
- BigQuery for analytics (unmatched)
- Favorable egress pricing (vs AWS)

**Cost Model**:
```
Compute (GKE): 2,000 vCPU × $0.08/hr × 730hr = $116,800/mo
GPU (50× A100): 50 × $2.95/hr × 730hr = $107,675/mo
Storage (1PB): 1,000TB × $0.020/GB = $20,000/mo
Egress (500TB/mo): 500TB × $0.08/GB = $40,000/mo
Cloud SQL: $8,000/mo
CDN: 100TB × $0.08/GB = $8,000/mo

Total: $300,475/mo = $3.6M/year
```

**Revenue per $1 OpEx**: $3.71B / $593M = **$6.25**

#### Amazon Web Services (AWS)

**Strengths**:
- Largest geographic footprint (32 regions)
- GovCloud (required for defense contracts)
- Mature ecosystem (more 3rd party integrations)
- S3 standard for object storage

**Weaknesses**:
- **Most expensive egress** ($0.12/GB vs $0.08 GCP)
- EC2 pricing 15-20% higher than GCP equivalent
- GPU availability poor (p4d instances scarce)

**Cost Model**:
```
Compute (EKS): 2,000 vCPU × $0.10/hr × 730hr = $146,000/mo
GPU (50× p4d.24xlarge): 50 × $32.77/hr × 730hr = $1,196,105/mo (!!!)
  ^ This is prohibitive — must use fewer GPUs or negotiate
  ^ Revised: 20× p4d = $478,442/mo
Storage (S3): 1PB × $0.023/GB = $23,000/mo
Egress: 500TB × $0.12/GB = $60,000/mo
RDS: $10,000/mo
CloudFront CDN: 100TB × $0.085/GB = $8,500/mo

Total: $726,000/mo = $8.7M/year (with fewer GPUs!)
```

**Revenue per $1 OpEx**: $3.71B / $870M = **$4.27** (31% worse than GCP)

**Verdict**: AWS is **40% more expensive** for GPU-heavy workloads.

#### Microsoft Azure

**Strengths**:
- Best for enterprise integration (AD, Office 365)
- Strong compliance (FedRAMP High, DoD IL6)
- Competitive GPU pricing (NCv3 series)
- Good for hybrid cloud (Azure Arc)

**Weaknesses**:
- Network performance inconsistent
- Kubernetes (AKS) less mature than GKE
- Storage egress pricing complex

**Cost Model**:
```
Compute (AKS): 2,000 vCPU × $0.096/hr × 730hr = $140,160/mo
GPU (50× NCv3): 50 × $3.06/hr × 730hr = $111,690/mo
Storage (Blob): 1PB × $0.018/GB = $18,000/mo
Egress: 500TB × $0.087/GB = $43,500/mo
Azure SQL: $9,000/mo
CDN: 100TB × $0.081/GB = $8,100/mo

Total: $330,450/mo = $4.0M/year
```

**Revenue per $1 OpEx**: $3.71B / $480M = **$7.73** (best!)

**But**: Azure has **reliability concerns** (more outages than GCP/AWS historically).

### Multi-Cloud Strategy (Recommended)

**Allocation**:
- **GCP (60%)**: Primary compute, analytics, GPU inference
- **AWS (25%)**: GovCloud, geographic coverage (APAC, MENA)
- **Azure (15%)**: Enterprise customers (AD integration), compliance

**Blended Cost**:
```
GCP: $3.6M × 0.60 = $2.16M
AWS: $8.7M × 0.25 = $2.18M (but negotiated down to $1.5M with EDPs)
Azure: $4.0M × 0.15 = $0.60M

Total: $4.26M/mo = $51M/year (pure cloud portion of hybrid)
```

**vs Single-Cloud**:
- GCP only: $43M/year (cheapest single cloud)
- Multi-cloud: $51M/year (+18% cost)
- **But**: Eliminates $500M-2B tail risk from vendor lock-in

**Multi-Cloud Premium**: 18% higher cost for 90% reduction in catastrophic risk.

## Part 3: Edge Economics Deep Dive

### The Unit Economics of Owned Infrastructure

**Per-Node Analysis (Tower Edge Node)**:

**CapEx (one-time)**:
```
Hardware:
- 2× NVIDIA L40S GPU: $14,000
- Server chassis (1U): $2,500
- NVMe 2TB: $800
- Networking (10GbE): $1,200
- Power supply: $500
- Installation: $1,500

Total CapEx: $20,500
Amortized (3yr): $6,833/year = $569/month
```

**OpEx (monthly)**:
```
Power (8kW × $0.12/kWh × 730hr): $700/mo
Connectivity (fiber 10Gbps): $150/mo
Maintenance (5% of CapEx annually): $85/mo
Pole lease: $50/mo

Total OpEx: $985/mo
```

**Total Cost of Ownership**: $569 (amort) + $985 (opex) = **$1,554/mo**

**Revenue per Node**:
```
Compute rental (GPU time): $1,200/mo
Bandwidth offset (Starlink): $400/mo
ShadowTag verification fees: $300/mo
PNT services: $100/mo

Total Revenue: $2,000/mo
```

**Gross Profit per Node**: $2,000 - $1,554 = **$446/mo** (**28.6% margin** at node level)

**But at scale** (10,000 nodes):
- Bulk purchasing: -15% CapEx ($17,425 per node)
- Maintenance automation: -30% OpEx ($690/mo)
- Total cost: **$1,181/mo**
- **Gross margin: 41%** at scale

**vs GCP Equivalent**:

To match 10,000 edge nodes in GCP:
```
10K nodes × 2× L40S equivalent = 20,000 GPU instances
GCP A100 (closest): $2.95/hr
Monthly: 20,000 × $2.95 × 730 = $43M/month

vs Owned: 10,000 × $1,181 = $11.8M/month

Savings: $31.2M/month = $374M/year
```

**Edge ownership saves $374M/year** at 10K node scale.

**NPV of Edge Investment** (10K nodes, 5-year horizon, 12% discount):

```
Year 0: -$174M (CapEx)
Year 1: +$95M (gross profit)
Year 2: +$142M (utilization improves)
Year 3: +$187M (full utilization)
Year 4: +$210M (margin expansion)
Year 5: +$245M + $52M residual value

NPV = -$174M + $95M/1.12 + $142M/1.12² + ... = $412M
IRR = 68%
```

**Edge infrastructure pays for itself in 18 months** and generates **$412M NPV**.

## Part 4: Kubernetes & Orchestration Costs

### GKE (Google Kubernetes Engine)

**Pricing Model**:
```
Cluster management: $0.10/hr per cluster = $73/mo
  (Free for first cluster under certain conditions)

Node costs:
- n2-standard-8 (8 vCPU, 32GB): $0.39/hr
- 100 nodes × $0.39 × 730hr = $28,470/mo

Autopilot mode (recommended):
- Per-pod pricing: $0.045/hr per vCPU
- 500 pods × 2 vCPU avg × $0.045 × 730 = $32,850/mo

GKE + node pool: ~$32k/mo for 100-node cluster
```

**Hidden Costs**:
- Persistent volume storage: +$5k/mo (50TB @ $0.10/GB)
- Load balancers: +$2k/mo (5× LBs @ $18/mo + traffic)
- Egress from pods: +$8k/mo (100TB internal → external)

**Total GKE**: $47k/mo base cluster

### EKS (Amazon Elastic Kubernetes Service)

**Pricing**:
```
Cluster management: $0.10/hr = $73/mo per cluster
Node costs (m5.2xlarge):
- 100 nodes × $0.384/hr × 730hr = $28,032/mo

EBS storage (gp3): 50TB × $0.08/GB = $4,000/mo
ALB load balancers: 5 × $23/mo = $115/mo
Egress: 100TB × $0.09/GB = $9,000/mo

Total EKS: $41k/mo
```

**Note**: EKS slightly cheaper but less performant than GKE.

### Self-Hosted Kubernetes (on owned edge)

**CapEx** (one-time):
```
Control plane nodes (3× high-availability):
- 3× servers ($8k each) = $24,000
- Amortized (3yr): $667/mo
```

**OpEx**:
```
Power (3× 500W): $130/mo
Networking: $200/mo
Maintenance (engineer time): $2,000/mo

Total: $3,000/mo
```

**Cost comparison** (100-node cluster):
- GKE: $47k/mo
- EKS: $41k/mo
- Self-hosted: **$3k/mo** (93% cheaper!)

**But**:
- Self-hosted requires 2-3 full-time SREs ($300k-450k/year = $37k/mo)
- **Break-even**: When managing >10 clusters (saves $440k/mo, costs $37k/mo in labor)

**Recommendation**:
- Use GKE for first 5 clusters (simplicity)
- Self-host when >10 clusters (economics flip)

**Savings vs GCP**: $14.4M/year
**Savings vs AWS**: $18M/year

**This single line item justifies edge ownership.**
