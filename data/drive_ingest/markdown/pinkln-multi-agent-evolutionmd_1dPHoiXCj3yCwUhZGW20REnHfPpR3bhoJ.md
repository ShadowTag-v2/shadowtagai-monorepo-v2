## Kernel-Chaining Framework

### What Is Kernel-Chaining?

**Kernel-Chaining** = Modular composition of reasoning strategies

Instead of monolithic prompts, chain specialized "kernels":

```
Input → [CoT Kernel] → [ToT Kernel] → [RCR Kernel] → [RTF Kernel] → Output

Each kernel:
- Has a specific reasoning style
- Can be swapped/evolved independently
- Rated by Glicko-2
- Optimized via GRPO
```

### Available Kernels

| Kernel | Full Name | Purpose | When to Use |
|--------|-----------|---------|-------------|
| **CoT** | Chain-of-Thought | Sequential reasoning | Linear problems |
| **ToT** | Tree-of-Thought | Explore alternatives | Multiple paths |
| **RCR** | Recursive Critique & Refine | Self-improvement | Iterative refinement |
| **RTF** | Reflect-Tag-Format | Structured output | Documentation |
| **TAG** | Task-Action-Goal | Planning | Multi-step tasks |
| **BAB** | Build-a-Better | Continuous improvement | Optimization |
| **CARE** | Context-Action-Result-Eval | Decision-making | High-stakes |
| **RISE** | Reason-Implement-Synthesize-Evolve | Complete workflow | End-to-end |

## GRPO vs PPO Training

### Algorithm Comparison

| Aspect | PPO (Traditional) | GRPO (Pinkln) | Advantage |
|--------|-------------------|---------------|-----------|
| **Optimization** | Single policy improvement | Group relative advantages | GRPO: Better multi-agent |
| **Reward Signal** | Absolute reward | Relative to group (G=8) | GRPO: More stable |
| **Compute** | 1 agent per batch | G agents per batch | PPO: Cheaper (but worse) |
| **Convergence** | Clipped loss (ε=0.2) | Relative advantage variance | GRPO: Faster for debates |
| **Use Case** | Single-agent tasks | Multi-agent reasoning | GRPO: Pinkln specialty |

### GRPO Training Loop (Simplified)

```python
# Pseudocode for GRPO training

def grpo_train(agents, group_size=8, epochs=3):
    """
    Group Relative Policy Optimization
    Trains agents by comparing within groups
    """
    for epoch in range(epochs):
        # 1. Sample G agents from pool
        group = random.sample(agents, group_size)

        # 2. Each agent generates response
        responses = [agent.generate(prompt) for agent in group]

        # 3. Get rewards (benchmark scores)
        rewards = [benchmark(r) for r in responses]

        # 4. Compute relative advantages
        mean_reward = np.mean(rewards)
        advantages = [r - mean_reward for r in rewards]

        # 5. Update policies proportional to advantage
        for agent, adv in zip(group, advantages):
            loss = -adv * log_prob(agent, response)
            agent.update(loss)

        # 6. Update Glicko-2 ratings
        for agent, reward in zip(group, rewards):
            glicko_update(agent, reward, group)

    return agents  # Evolved via group competition
```

**Key Insight**: Agents learn by comparing to peers, not absolute rewards

### Compute Requirements

**PPO Training** (baseline):
- 1× A100 per training run

**GRPO Training** (Pinkln):
- 4× A100 per training run (G=8 agents, 2 per GPU)
- Batch size: 8 (one per agent)
- Time: 6 hours
- **4× cost but better multi-agent performance**

## Glicko-2 Agent Ratings

### Why Glicko-2 Instead of Elo?

| Feature | Elo | Glicko-2 | Winner |
|---------|-----|----------|--------|
| **Rating Deviation** | No | Yes (uncertainty) | Glicko-2 |
| **Volatility** | No | Yes (consistency) | Glicko-2 |
| **Inactive Penalty** | No | Yes (RD increases) | Glicko-2 |
| **Simplicity** | Simple | Complex | Elo |
| **Accuracy** | Good | Better | Glicko-2 |

**Pinkln Choice**: Glicko-2 for sophisticated agent ranking

### Glicko-2 Implementation

```python
# glicko2.py - Agent rating system

class Glicko2Player:
    """
    Glicko-2 player for agent ratings

    Attributes:
        mu: Rating (default 1500)
        phi: Rating deviation (uncertainty)
        vol: Volatility (rating consistency)
    """

    def __init__(self, mu=1500, phi=350, vol=0.06):
        self.mu = mu      # Rating
        self.phi = phi    # Deviation (uncertainty)
        self.vol = vol    # Volatility (consistency)

    def get_rating(self):
        """Get current rating"""
        return self.mu

    def get_rd(self):
        """Get rating deviation"""
        return self.phi

    def get_vol(self):
        """Get volatility"""
        return self.vol

    def update(self, results, tau=0.5, tol=1e-6):
        """
        Update rating based on match results

        Args:
            results: List of (opponent, score) tuples
                     score: 1 (win), 0.5 (draw), 0 (loss)
            tau: System constant (volatility change)
            tol: Convergence tolerance

        Returns:
            Updated mu, phi, vol
        """
        # Convert to Glicko-2 scale
        mu_ = (self.mu - 1500) / 173.7178
        phi_ = self.phi / 173.7178

        # Compute variance
        v = self._compute_v(results, mu_)

        # Compute delta (improvement)
        delta = self._compute_delta(results, mu_, v)

        # Update volatility
        self.vol = self._update_volatility(
            delta, v, phi_, tau, tol
        )

        # Update rating deviation
        phi_star = np.sqrt(phi_**2 + self.vol**2)
        phi_new = 1 / np.sqrt(1/phi_star**2 + 1/v)

        # Update rating
        mu_new = mu_ + phi_new**2 * sum([
            self._g(opp.get_rd() / 173.7178) *
            (s - self._E(mu_, mu_opp, phi_opp))
            for opp, s in results
        ])

        # Convert back to Glicko scale
        self.mu = mu_new * 173.7178 + 1500
        self.phi = phi_new * 173.7178

        return self.mu, self.phi, self.vol

    def _compute_v(self, results, mu_):
        """Compute variance"""
        return 1 / sum([
            self._g(opp.get_rd() / 173.7178)**2 *
            self._E(mu_, (opp.get_rating() - 1500) / 173.7178,
                    opp.get_rd() / 173.7178) *
            (1 - self._E(mu_, (opp.get_rating() - 1500) / 173.7178,
                         opp.get_rd() / 173.7178))
            for opp, _ in results
        ])

    def _compute_delta(self, results, mu_, v):
        """Compute rating improvement"""
        return v * sum([
            self._g(opp.get_rd() / 173.7178) *
            (s - self._E(mu_, (opp.get_rating() - 1500) / 173.7178,
                         opp.get_rd() / 173.7178))
            for opp, s in results
        ])

    def _update_volatility(self, delta, v, phi_, tau, tol):
        """Update volatility using iterative method"""
        # Illinois algorithm for volatility
        a = np.log(self.vol**2)

        def f(x):
            ex = np.exp(x)
            num1 = ex * (delta**2 - phi_**2 - v - ex)
            den1 = 2 * (phi_**2 + v + ex)**2
            num2 = x - a
            den2 = tau**2
            return num1 / den1 - num2 / den2

        A = a
        if delta**2 > phi_**2 + v:
            B = np.log(delta**2 - phi_**2 - v)
        else:
            k = 1
            while f(a - k * tau) < 0:
                k += 1
            B = a - k * tau

        fA, fB = f(A), f(B)

        while abs(B - A) > tol:
            C = A + (A - B) * fA / (fB - fA)
            fC = f(C)

            if fC * fB <= 0:
                A, fA = B, fB
            else:
                fA /= 2

            B, fB = C, fC

        return np.exp(A / 2)

    @staticmethod
    def _g(phi):
        """Glicko-2 g function"""
        return 1 / np.sqrt(1 + 3 * phi**2 / np.pi**2)

    @staticmethod
    def _E(mu, mu_j, phi_j):
        """Expected score against opponent"""
        return 1 / (1 + np.exp(-Glicko2Player._g(phi_j) * (mu - mu_j)))

# Usage Example: Rate agents after DTE round

# Initialize agents
deep_reasoning = Glicko2Player(mu=2000, phi=200, vol=0.06)
code_crafter = Glicko2Player(mu=1950, phi=180, vol=0.05)
wealth_accel = Glicko2Player(mu=2100, phi=220, vol=0.07)

# DTE results (benchmark scores converted to win/loss)
# deep_reasoning beat code_crafter on HumanEval
deep_reasoning.update([(code_crafter, 1.0)])  # Win
code_crafter.update([(deep_reasoning, 0.0)])  # Loss

# wealth_accel beat deep_reasoning on revenue optimization
wealth_accel.update([(deep_reasoning, 1.0)])
deep_reasoning.update([(wealth_accel, 0.0)])

print(f"Deep Reasoning: {deep_reasoning.get_rating():.0f} ± {deep_reasoning.get_rd():.0f}")
print(f"Code Crafter: {code_crafter.get_rating():.0f} ± {code_crafter.get_rd():.0f}")
print(f"Wealth Accelerator: {wealth_accel.get_rating():.0f} ± {wealth_accel.get_rd():.0f}")
```

### Viral Leverage Model

**Concept**: Maximize organic growth coefficient (k-factor)

**Formula**:
```
k = i × c
where:
  i = invites sent per user
  c = conversion rate of invites

k > 1 = viral growth
k < 1 = paid growth required
```

**Wealth Accelerator Strategy**:

1. **Measure current k**:
   ```python
   invites_per_user = 2.3
   invite_conversion = 0.18
   k_current = 2.3 * 0.18 = 0.414  # Sub-viral
   ```

2. **Identify levers**:
   - **Increase i**: Referral incentives, share prompts, network effects
   - **Increase c**: Landing page for referrals, social proof, urgency

3. **GRPO-optimize interventions**:
   - Test 8 variants (G=8) of referral copy
   - Measure lift in k-factor
   - Evolve winning strategies

4. **Target**:
   ```python
   k_target = 1.2  # 20% organic growth
   required_lift = 1.2 / 0.414 = 2.9× improvement

   Path:
   - i: 2.3 → 3.5 (+52% via incentives)
   - c: 0.18 → 0.34 (+89% via optimized landing)
   - k: 0.414 → 1.19 = viral!
   ```
