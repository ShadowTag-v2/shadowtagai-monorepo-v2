# Thumbly SaaS Design System & Architecture

## Core Identity
Thumbly is an instantaneous YouTube thumbnail generator utilizing Stripe-deterministic ledgers and extremely high-impact visual aesthetics to drive conversion-rate perfection. Under the hood, it rigidly enforces the utilization of **Nano Banana 2 (Gemini 3.1 Flash Image)** for primary text-rendering and world-knowledge asset generation at maximum inference speed.

## Intelligent Deployment Target
1. **Frontend Rendering**: Stitch AI Engine (Direct synthesis via this `DESIGN.md`).
2. **Backend/Cloud Infrastructure**: Google AI Studio native `@Firebase` Integration (Firestore DB, Firebase Authentication, Cloud Functions). Local Next.js/Supabase routes are officially deprecated in favor of this hyper-tight bundle.

## Aesthetic Mandate: Dark Luxury Glassmorphism
All UI elements generated by Stitch must adhere strictly to these physical principles:
- **Spatial Backgrounds**: Absolute pure black base `#000000` overlaid with highly diffused, fluid `framer-motion` generated emerald and cyan radial blurs (e.g., `bg-emerald-500/20 blur-[120px]`).
- **Glass Surfaces**: Translucent `bg-zinc-900/40` or abstract `bg-black/40` surfaces strictly layered with `backdrop-blur-xl` properties.
- **Micro-Borders**: Sub-pixel thin, high-contrast borders `border-white/10`. Hover interactions must linearly ramp luminosity to `border-white/20`.
- **Typography Density**: Geometric fonts (Inter / Outfit / system-ui). Hero headers must utilize radical gradient spans `bg-clip-text text-transparent bg-gradient-to-r from-emerald-400 to-cyan-400`.

## Functional Viewport Map
1. **Unauthenticated Hero Screen**:
   - H1: "Generate High-Conversion YouTube Thumbnails."
   - Prominent, illuminated Call-to-Action routing directly into the Firebase OAuth gate.
2. **Authenticated Studio Sandbox**:
   - The central infinite-canvas representing the 1280x720 rendering boundaries.
   - Text input vectors enforcing prompt injection constraints.
3. **Entitlement Hooks (The Ledger)**:
   - Active Firebase function calls verifying `userRecord.credits > 0` before any visual synthesis executes. Generation is explicitly blocked upstream on zero-balance accounts.
