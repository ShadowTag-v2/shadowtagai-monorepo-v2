### 2a. Action Verb Analysis (California Bar Method)

Each action verb absolutely gets its own, separate consideration.

**Process:**

1. IDENTIFY all action verbs in the requirement/problem

2. ISOLATE each verb into its own sentence

3. ANALYZE each verb independently:

   - What triggers it?

   - What inputs?

   - What outputs?

   - What can fail?

   - What edge cases?

4. SYNTHESIZE findings back together

**Example Decomposition:**

```

"System receives, validates, transforms, stores, sends data"


1. RECEIVES → [source, format, auth, errors]

2. VALIDATES → [rules, sanitization, messages]

3. TRANSFORMS → [mapping, loss, edge cases]

4. STORES → [location, schema, transactions]

5. SENDS → [recipient, channel, failures]

```

This IS obsessing over details—you cannot obsess without decomposition.

## Plan Mode Style (Cor.32)

### Core Philosophy

"Sacrifice grammar for concision. Surface decisions last."

### Rules


- Concision > Grammar: Every token carries action, state, or decision

- Line = atomic instruction: One line, one outcome. No conjunctions

- All verbs imperative: "Add param," "Refactor func," "Defer X to Phase 2"

- Readable at a glance: Grasp entire plan in <5 seconds

- End with open questions or blockers—never buried

### Structure

```

<package or module>:

- <action> → <target>

- <next action> → <target>

Impl:

- <approach summary>

- <temporary compat plan>

- <phase boundaries>

Unresolved Qs:

- <open decision 1>?

- <open decision 2>?

```

### Symbols


- → transformation/migration

- = assignment/decision

- + add/include

- - remove/deprecate

- ? question/unresolved

- ⚠️ risk/dependency

- // inline note

### Verb Decomposition Rule

Before planning, decompose the task by action verbs:

```

Task: "System receives, validates, transforms, stores, sends data"

Verb Decomposition:

- RECEIVES → [analysis of inputs, auth, errors]

- VALIDATES → [analysis of rules, sanitization]

- TRANSFORMS → [analysis of mapping, edge cases]

- STORES → [analysis of schema, transactions]

- SENDS → [analysis of channels, failures]

Then Plan Mode each verb's implementation:

receives/:

- add HTTP POST endpoint → /api/data

- validate Content-Type → JSON only

- implement rate limiting → 100/min

validates/:

- add Pydantic schema → DataInput

- sanitize strings → XSS prevention

- return 422 → with field errors

```

Each verb becomes its own implementation phase with atomic actions.
