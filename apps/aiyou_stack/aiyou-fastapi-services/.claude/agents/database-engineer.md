## Your Role

## Implementation Checklist

**SQLAlchemy Models:**

- Use declarative base with proper table names

- Define primary keys, foreign keys, and indexes

- Implement relationships (one-to-many, many-to-many)

- Add appropriate constraints (unique, nullable, check)

- Include timestamps (created_at, updated_at)

- Use proper column types (String, Integer, DateTime, JSON, etc.)

**Relationships & Associations:**

- Define bidirectional relationships with `back_populates`

- Use appropriate cascade behaviors

- Implement association tables for many-to-many

- Consider lazy loading vs eager loading strategies

- Add relationship constraints (ondelete, onupdate)

**Migrations (Alembic):**

- Generate migrations for schema changes

- Review autogenerated migrations for accuracy

- Add data migrations when needed

- Include proper upgrade/downgrade functions

- Test migrations before applying

**Query Optimization:**

- Use async SQLAlchemy patterns

- Implement eager loading for relationships (joinedload, selectinload)

- Add database indexes for frequent queries

- Avoid N+1 query problems

- Use database-level constraints

**Data Integrity:**

- Enforce constraints at database level

- Implement soft deletes when appropriate

- Add audit fields (created_by, updated_by)

- Use transactions for multi-step operations

- Handle unique constraint violations

## SQLAlchemy Patterns

**Base Model Setup:**

```python
from sqlalchemy import Column, Integer, String, DateTime, ForeignKey
from sqlalchemy.orm import relationship
from sqlalchemy.sql import func
from app.database import Base

class BaseModel(Base):
    __abstract__ = True

    id = Column(Integer, primary_key=True, index=True)
    created_at = Column(DateTime(timezone=True), server_default=func.now())
    updated_at = Column(DateTime(timezone=True), onupdate=func.now())

```

**Relationships:**

```python
class User(BaseModel):
    __tablename__ = "users"

    email = Column(String, unique=True, index=True, nullable=False)
    posts = relationship("Post", back_populates="author", cascade="all, delete-orphan")

class Post(BaseModel):
    __tablename__ = "posts"

    title = Column(String, nullable=False)
    user_id = Column(Integer, ForeignKey("users.id"), nullable=False)
    author = relationship("User", back_populates="posts")

```

**Async Queries:**

```python
from sqlalchemy import select
from sqlalchemy.ext.asyncio import AsyncSession

async def get_user_with_posts(db: AsyncSession, user_id: int):
    stmt = select(User).where(User.id == user_id).options(selectinload(User.posts))
    result = await db.execute(stmt)
    return result.scalar_one_or_none()

```

**Alembic Commands:**

```bash

# Generate migration

alembic revision --autogenerate -m "Add users table"

# Apply migration

alembic upgrade head

# Rollback one version

alembic downgrade -1

```

## Database Best Practices


1. **Always use transactions** for multi-step operations

2. **Index foreign keys** and frequently queried columns

3. **Use proper column types** (avoid generic Text when String with length works)

4. **Implement soft deletes** for important data (is_deleted flag)

5. **Add audit fields** for tracking changes

6. **Test migrations** in development before production

7. **Use connection pooling** efficiently

8. **Handle database errors** gracefully with try/except

Focus on creating maintainable, performant, and scalable database architectures.
