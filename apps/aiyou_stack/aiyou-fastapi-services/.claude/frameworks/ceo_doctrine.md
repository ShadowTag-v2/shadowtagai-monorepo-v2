# CEO Doctrine Integration

> **Claude Code Quick Reference** - Apply founder doctrine to AI-assisted decisions.

## JR Engine (Every Major Decision)

```
PURPOSE  → Does this advance Pnkln / generational wealth?
REASONS  → Defensible to critical board / court?
BRAKES   → Worst-case p99 survivable?

Any "No" → Redesign or reduce commitment
```

## Economic Thresholds

| Metric | Target | Kill Threshold |
|--------|--------|----------------|
| ROI | ≥3× in 18 months | <3× and not improving |
| LTV:CAC | ≥4:1 in 12-18 months | <2:1 and not improving |
| NPS | >40 core users | <20 |
| "Very Disappointed" | ≥70% | <50% |

## Risk Matrix (ATP 5-19)

```
Probability: A (frequent) → E (rare)
Severity:    I (catastrophic) → IV (minor)

EH/H → redesign before proceeding
M    → proceed with mitigation
L    → accept and monitor
```

---

## Seven Powers Checklist

When evaluating any product/feature:

- [ ] **Data Flywheel** - Does usage improve the product?
- [ ] **Scale Economies** - Do unit costs decrease?
- [ ] **Switching Costs** - Is it painful to leave?
- [ ] **Network Effects** - Does value increase per user?
- [ ] **Cornered Resource** - Exclusive access to key asset?
- [ ] **Process Power** - Superior operations?
- [ ] **Brand Power** - Trust/recognition premium?

At least ONE required for defensible product.

## PDS Test (First-in-Field)

Before any launch:

| Test | Question | Required |
|------|----------|----------|
| **P**ractical | Actually usable, not science project? | Yes |
| **D**esirable | Users strongly want it? | Yes |
| **S**calable | Unit economics improve with scale? | Yes |

## Speed Loop Priority

```
Loop Time = idea → ship → measure → learn

MINIMIZE THIS. Always.
```

---

## Decision Quick-Check

Before committing significant resources:

1. **Purpose**: Advances generational wealth?
2. **ROI**: ≥3× path visible?
3. **Risk**: EH/H risks mitigated?
4. **Moat**: Building which power?
5. **PMF**: 10× better for specific segment?
6. **Speed**: Shortens our loops?
7. **Logged**: In decision log?
