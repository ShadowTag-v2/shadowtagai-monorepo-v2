# Template 02: Design Critique (Jobs-Style)

## Purpose
Get ruthless, specific design feedback with focus on beauty, simplicity, and function.

## When to Use
- Reviewing UI/UX designs
- Simplifying complex interfaces
- Validating design decisions
- Getting honest design assessment

---

## Copy-Paste Prompt

```
@design-critique [DESIGN ARTIFACT]

Critique this design with Jobs-level ruthlessness:

**Design Type**: [UI / UX / Visual / Interaction / Information Architecture]
**Design Artifact**: [URL, file path, or description]
**Platform**: [Web / Mobile / Desktop / Cross-platform]

**Evaluation Criteria**:
- Beauty (visual appeal, aesthetics)
- Simplicity (unnecessary complexity?)
- Function (does it work effortlessly?)
- Delight (does it surprise and please?)
- Coherence (unified vision?)

**Ruthlessness Level**: [Gentle / Balanced / Ruthless / Jobs-Level]

**Specific Questions**:
1. [Question 1 about design]
2. [Question 2 about design]

**Context**:
- Target User: [Who will use this]
- Primary Use Case: [What they're trying to do]
- Brand Guidelines: [If any]

**Deliverables**:
- Beauty score (1-10) with rationale
- Simplicity score (1-10) with rationale
- Function score (1-10) with rationale
- What works (celebrate excellence)
- What fails (be brutally specific)
- Remove immediately (list of deletions)
- Design debt (future problems)
- Path to greatness (3-5 improvements)
- Final verdict (Approve/Revise/Reject)
```

---

## Example Usage

```
@design-critique ./mockups/dashboard-v3.png

Critique this design with Jobs-level ruthlessness:

**Design Type**: UI
**Design Artifact**: Dashboard design for SaaS analytics platform
**Platform**: Web

**Evaluation Criteria**:
- Beauty (visual appeal, aesthetics)
- Simplicity (unnecessary complexity?)
- Function (does it work effortlessly?)
- Delight (does it surprise and please?)
- Coherence (unified vision?)

**Ruthlessness Level**: Jobs-Level

**Specific Questions**:
1. Does the visual hierarchy guide the eye to the most important metrics?
2. Are there any elements that don't earn their place on the screen?
3. Could a first-time user understand this dashboard in 5 seconds?

**Context**:
- Target User: Technical founders and product managers
- Primary Use Case: Quick health check of product metrics
- Brand Guidelines: Minimal, data-focused, professional

**Deliverables**:
- Beauty score (1-10) with rationale
- Simplicity score (1-10) with rationale
- Function score (1-10) with rationale
- What works (celebrate excellence)
- What fails (be brutally specific)
- Remove immediately (list of deletions)
- Design debt (future problems)
- Path to greatness (3-5 improvements)
- Final verdict (Approve/Revise/Reject)
```

---

## Tips for Best Results

1. **Provide Context**: Share user persona and use case
2. **Be Specific**: Ask targeted questions about design choices
3. **Set Ruthlessness**: Jobs-Level for brutal honesty, Gentle for constructive
4. **Include Constraints**: Mention technical or brand limitations
5. **Attach Visuals**: Screenshots, Figma links, or detailed descriptions
