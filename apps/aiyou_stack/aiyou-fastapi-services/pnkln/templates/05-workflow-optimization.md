# Template 05: Workflow Optimization & Simplification

## Purpose
Ruthlessly simplify processes, eliminate waste, and automate repetitive work.

## When to Use
- Process is too slow
- Too many manual steps
- Bottlenecks identified
- Team frustrated with workflow
- Need to scale without hiring

---

## Example Usage

```
Optimize this workflow with ruthless simplification:

**Workflow Description**:
Current feature development process:
1. PM writes spec in Notion (2 hours)
2. Designer creates mockups in Figma (4 hours)
3. PM and Designer review meeting (30 min)
4. Revisions to mockups (2 hours)
5. Engineering review meeting (30 min)
6. Engineer implements feature (8 hours)
7. Manual testing by PM (1 hour)
8. Bug fixes (2 hours)
9. Code review (1 hour)
10. Deploy to staging (30 min)
11. PM approval (30 min)
12. Deploy to production (30 min)

Total: ~22 hours, 4 meetings, 5 people involved

**Workflow Type**: Development

**Current Metrics**:
- Average Cycle Time: 4 days (from spec to production)
- Error Rate: 20% (features need bug fixes)
- Throughput: 3-4 features per month
- Team Size: 1 PM, 1 Designer, 2 Engineers
- Time Spent per Week: ~88 hours total team time

**Known Pain Points**:
1. Too many handoffs and meetings
2. Rework due to miscommunication
3. Manual testing catches bugs late
4. Deployment process is manual and error-prone
5. Context switching between multiple features

**Optimization Goals**:
- [X] Reduce time (target: 50% reduction)
- [X] Reduce errors (target: 80% reduction)
- [X] Increase throughput (target: 100% increase)
- [X] Reduce manual work (target: 40% automation)

**Available Tools/Systems**:
- GitHub (code, CI/CD)
- Figma (design)
- Notion (specs)
- Linear (issue tracking)
- Vercel (hosting)

**Constraints**:
- Can't hire more people
- Must maintain quality
- Max $500/month for new tools

**Deliverables**:
1. Current state analysis (waste inventory, bottlenecks)
2. Optimized workflow (streamlined process)
3. Automation roadmap (quick wins → long-term)
4. Before/after metrics (expected improvements)
5. Implementation plan (weekly milestones)
```

## Tips for Best Results

1. **Map Current State First**: Document every step before optimizing
2. **Quantify Pain**: Use metrics (hours wasted, error rates)
3. **Involve the Team**: They know where the waste is
4. **Start with Quick Wins**: Build momentum with easy automations
5. **Measure Before/After**: Track actual improvements
