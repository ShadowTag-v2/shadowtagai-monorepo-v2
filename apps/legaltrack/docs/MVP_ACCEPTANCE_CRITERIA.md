## 1. Zero-Trust Security & Compliance (Non-Negotiable Gates)
The architecture must protect against the devastating "Tea app" failure class.
* **[ ] SC.1 (Zero Plaintext):** Execution is halted and PR rejected if any database (RDS, CloudSQL/pgvector) or transit path operates without TLS and KMS-backed encryption.
* **[ ] SC.2 (Immutable Backups):** All critical workflow states and ingested emails must sync to S3-class object storage with active Object Lock preventing deletion.
* **[ ] SC.3 (NY SB S7263 / UPL Compliance):** The UI and underlying agents **shall not generate or transmit synthesized legal advice**, opinions, or qualitative analyses of case law directly to end users. The UI acts *strictly* as a procedural router, presenting calculated dates and citing the objective source text (e.g., FRCP 12a), exposing no liability under New York's AI private right of action.

## 2. Core Capabilities (The 5 MVP Pillars)

### Pillar A: Ingestion Engine
* **[ ] IN.1 (Secure Webhook):** The system must securely ingest filing events via email payloads (Gmail/Outlook webhooks over HTTPS).
* **[ ] IN.2 (Pnkln Kernel-Chaining):** Extraction latency must remain under 100ms via the 3-Kernel Spec, reducing token usage by 98.5%.

### Pillar B: Live Rules DB (Jurisdiction Engine)
* **[ ] RD.1 (Deterministic Routing):** System must parse and map extracted filing metadata to the correct local/state/federal rule baseline (e.g., CA vs. NY) using the Cheat Sheet Fusion contextual reduction.

### Pillar C: Dynamic Timeline Generator
* **[ ] TL.1 (Idempotent Sync):** Must map calculated deadlines deterministically to the database and establish a bidirectional, idempotent sync with Google/Outlook calendars (Zero Duplication).

### Pillar D: Configurable Enforcement Slider
* **[ ] EF.1 (Escalating Intensity):** The Schiznit / CEOTrack Prodding engine must execute API-driven ambient nudges (e.g., Tesla cabin pre-conditioning, SMS, desktop alerts) based on user-configured intensities ("gentle" to "no-slack").
* **[ ] EF.2 (Cross-Industry Scalability):** The core enforcement algorithm must remain abstracted such that it easily crosses over into Tax/Accounting, Healthcare Admin, and Securities without core rewrites.

### Pillar E: Cross-Platform Dark Luxury UI
* **[ ] UI.1 (Mobile Critical Tiles):** Mobile applications must feature full-screen "Tap-to-Act" tiles featuring high-contrast typography, single-word directives ("FILE", "SIGN"), and escalating ambient alerts.
